### 开发目标

### 完成项目前期准备，搭建系统底层架构，为后续业务开发提供基础支撑。

### **核心任务**

### 需求分析与系统设计：明确校园组队场景的用户角色（普通用户 / 项目队长）、业务流程（注册 - 登录 - 组队 - 聊天）与核心功能边界。

### 数据库设计：设计用户表、项目表、申请表、消息表等核心数据表结构，定义字段约束与外键关联。

### 环境搭建：确定技术栈（FastAPI + SQLite + HTML/JS），搭建项目骨架，实现数据库自动初始化。

### **代码实现要点**

### 数据库初始化函数 init_db()：创建所有数据表，兼容后续表结构升级（通过ALTER TABLE异常捕获实现），项目启动时自动执行。

### 工具类封装：实现密码哈希函数hash_password()（SHA256 加密存储）、会话 ID 生成函数generate_session_id()，保障基础安全。

### 项目生命周期配置：通过lifespan上下文管理器，实现项目启动时自动初始化数据库。
