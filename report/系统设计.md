
## 三、系统设计 【中期必填】

### 1. 系统架构

系统采用前后端分层架构：

```text
┌─────────────────────────────────────────┐
│           浏览器端                       │
│  Jinja2模板 + HTML/CSS/JS + FetchAPI    │
└─────────────────┬───────────────────────┘
                  │
          HTTP / WebSocket
                  │
┌─────────────────▼───────────────────────┐
│          FastAPI 后端                   │
│    RESTful接口 + WebSocket（聊天）      │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│         SQLite 数据库                    │
└─────────────────────────────────────────┘
```

![系统架构图](/assets/系统架构图.png)


### 2. 模块设计

| 模块 | 功能说明 |
|-----|---------|
| **用户模块** | 注册/登录/互斥登录/个人资料管理 |
| **标签模块** | 技能/兴趣标签管理及匹配算法 |
| **招募模块** | 发布/编辑/隐藏/删除/状态更新 |
| **搜索推荐模块** | 多维度筛选 + 智能推荐逻辑 |
| **申请审批模块** | 入队申请、审批、成员管理 |
| **消息模块** | WebSocket聊天 + 未读红点 + 弹窗通知 |
| **群聊模块** | 自动创建项目群、成员同步管理 |

#### 主要数据表设计（核心表）

| 表名 | 说明 | 关键字段 |
|-----|------|---------|
| `users` | 用户表 | phone(主键), name, password, college, major, skills, session_id |
| `user_sessions` | 会话表 | session_id(主键), phone |
| `projects` | 招募项目表 | id(主键), title, leader_phone, description, tags, status |
| `applications` | 申请表 | id(主键), proj_id, applicant_phone, status |
| `messages` | 消息表 | id(主键), sender_phone, chat_type, target_id, content |
| `chat_state` | 聊天状态表 | phone, chat_type, target_id (联合主键) |

> 📌 **中期要求**：架构、模块、数据库设计基本定稿。
> 📌 **结项要求**：与最终系统一致，如有调整需同步更新。
