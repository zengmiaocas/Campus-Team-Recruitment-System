## 一、项目概述 【中期必填】

### 1. 项目背景

本项目来源于当前大学生在竞赛、考研、求职、课程项目等场景下"找搭子难、组队效率低"的实际痛点。

现有微信群、QQ群、校园墙等方式存在以下问题：
- 信息分散，缺乏统一发布平台
- 缺乏精准筛选与匹配机制
- 跨班级、跨专业沟通渠道不畅
- 组队过程缺乏规范的审批与管理流程

本系统提供一个集中发布招募、精准匹配、在线沟通的平台，有效降低组队门槛，提高组队成功率。

### 2. 系统目标

- ✅ 提供统一组队平台，降低跨班级、跨专业组队门槛
- ✅ 基于技能/兴趣标签实现精准匹配与智能推荐
- ✅ 支持招募发布、申请审批、在线聊天与消息通知
- ✅ 保障多端互斥登录与数据安全
- ✅ 支持Markdown富文本排版，提升招募信息专业度

### 3. 开发环境

| 分类 | 技术选型 |
|-----|---------|
| **后端** | Python 3.13 + FastAPI + Uvicorn |
| **前端** | Jinja2 + HTML5/CSS3 + 原生JavaScript (Fetch API) |
| **数据库** | SQLite (轻量级，免配置) |
| **开发工具** | VS Code / PyCharm + Git |
| **依赖管理** | pip |

## 四、系统实现

### 1. 关键技术

| 技术 | 应用场景 |
|------|----------|
| **FastAPI异步框架** | 构建RESTful API接口，支持异步请求处理与自动文档生成 |
| **SQLite轻量级数据库** | 数据持久化存储，免配置部署，支持多表关联查询 |
| **Session + Cookie会话管理** | 用户登录状态维护，生成唯一session_id存入cookie |
| **多端互斥登录** | 新设备登录时删除旧设备session，实现“顶号”强制下线 |
| **SHA256密码加密** | 用户密码存储前进行哈希加密，保障账号安全 |
| **轮询消息推送** | 前端每2.5秒轮询后端接口，实现消息实时通知与红点提醒 |
| **Markdown实时渲染** | 前端使用marked.js库将项目描述渲染为HTML |
| **Ajax异步表单提交** | 所有表单通过fetch API异步提交，无需刷新页面 |
| **智能推荐算法** | 基于用户技能标签与项目标签的模糊匹配（子串包含+大小写归一化） |

### 3. 核心代码片段

**多端互斥登录（顶号）实现：**

```python
@app.post("/do_login")
async def do_login(username: str = Form(...), password: str = Form(...)):
    new_sess = generate_session_id()
    with sqlite3.connect(DB_FILE, timeout=10) as conn:
        user = conn.cursor().execute(
            "SELECT phone FROM users WHERE (phone=? OR student_id=?) AND password=?",
            (username, username, hash_password(password))
        ).fetchone()
        if user:
            phone = user[0]
            # 关键：删除该用户所有旧session，实现互斥登录
            conn.execute("DELETE FROM user_sessions WHERE phone = ?", (phone,))
            conn.execute("INSERT INTO user_sessions (session_id, phone) VALUES (?, ?)", (new_sess, phone))
            conn.commit()
            res = RedirectResponse(url="/", status_code=303)
            res.set_cookie("session_token", new_sess, max_age=604800)
            return res
        return alert_and_redirect("❌ 账号或密码错误！", "/login")
```

**第一步：准备项目文件**

将以下文件放置于同一目录下：

打开终端（命令行），进入项目目录，执行以下命令：


**第三步：启动服务**

在终端中执行：

```bash
python main.py
```

启动成功后，控制台会显示：

## 2. 操作指南

### 2.1 注册与登录

**注册账号**

1. 访问首页，点击"没有账号？去注册"
2. 填写：学号（12位数字）、姓名/昵称、手机号、密码
3. 点击"注册并登录"完成注册

**登录系统**

- 方式一：使用手机号 + 密码登录
- 方式二：使用学号（12位）+ 密码登录

**修改密码**

1. 点击大厅右上角"修改密码"按钮
2. 输入学号、手机号进行身份验证
3. 输入原密码和新密码，确认后提交

### 2.2 完善个人信息

1. 点击大厅右上角的"个人信息"按钮（或头像）
2. 填写以下信息（带 * 为必填）：
   - 学院、专业、班级
   - 技能/兴趣标签（如：Python，Java，前端），用于智能推荐
   - QQ号、微信号（方便组队联系）
   - 个人荣誉（如：国家奖学金）
   - 个性签名
3. 点击"保存并关闭"

> 💡 提示：技能标签填写越详细，智能推荐越精准！

### 2.3 发布招募项目

1. 点击大厅顶部的"➕ 发布招募"按钮
2. 填写项目信息：
   - 项目名称
   - 已有人数 / 共需人数
   - 技术标签（逗号分隔，如：Python，数据库）
   - 项目描述（支持 Markdown 语法）
3. 点击"立即发布"，项目将显示在公共招募大厅

### 2.4 管理我的招募

点击"📝 我的招募"按钮

对每个项目可执行以下操作：

| 操作 | 说明 |
|------|------|
| 隐藏招募 | 仅自己可见，其他人看不到此项目 |
| 停止招募 | 项目状态变为"已满员/截止"，不可再申请 |
| 删除招募 | 永久删除项目（不可恢复） |

### 2.5 申请加入项目

1. 在公共招募大厅浏览项目
2. 点击项目卡片中的"申请加入"按钮
3. 队长审批前可点击"我的申请"中的"撤销申请"取消

### 2.6 队长审批申请

1. 点击"🛡️ 队长审批"按钮
2. 查看待审核成员列表，可看到申请者的荣誉信息
3. 选择"同意进组"或"婉拒"
4. 审批后，申请记录会保留在历史中
5. 当队伍满员时，项目状态自动变为"已截止"

### 2.7 消息聊天

**私聊功能**

1. 在大厅点击任意用户的头像
2. 点击"💬 发起聊天"
3. 输入消息后发送

**项目群聊**

- 入队成功后，可通过"我的申请"或项目卡片进入群聊
- 群聊内所有队员和队长均可收发消息
- 队长可在右侧成员面板中"移出"队员

**消息管理**

- 未读消息会在"消息"按钮和对应会话上显示红点
- 点击会话列表右侧的"🗑️"按钮可清空聊天记录（仅自己不可见）

### 2.8 搜索与推荐

**关键词搜索**

在搜索框中输入项目名称或描述关键词，点击"快速搜索"

**标签筛选**

在标签筛选框中输入标签（如：Python），按标签过滤项目

**智能推荐**

- 系统根据您填写的"技能/兴趣标签"自动匹配推荐项目
- 推荐结果显示在右侧"猜你喜欢"面板
- 匹配度越高，排名越靠前

### 2.9 退出与多端互斥

- 点击"退出"按钮安全登出
- 同一账号在另一设备登录时，原设备会被强制下线并弹出提示
