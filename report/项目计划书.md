# 项目计划书

## 一、项目名称

竞赛自由组队平台

## 二、项目内容与需求分析

### 1. 项目内容

本项目旨在设计并实现一个竞赛自由组队系统，解决学生在参加各类竞赛时寻找合适队友困难的问题。

系统主要研究内容包括：用户注册登录与个人名片管理、技能标签体系的构建与应用、招募大厅的动态发布与智能推荐、队长对队员申请的审批管理、以及在线聊天与消息实时通知机制。

通过上述功能模块的实现，为竞赛参与者提供一个高效、便捷的组队平台，降低组队门槛，提升组队成功率。

### 2. 需求分析

#### （1）研究背景

近年来，随着各类学科竞赛、创新创业大赛的不断增多，大学生参与竞赛的热情日益高涨。然而，在实际参赛过程中，许多学生面临“找不到合适队友”的困境：一方面，身边熟悉的朋友往往专业背景相似，难以形成技能互补；另一方面，跨班级、跨专业的同学之间缺乏有效的沟通渠道，导致许多有参赛意愿的学生因组队困难而放弃机会。

现有的组队方式多依赖微信群、QQ群、校园墙等社交平台，存在信息分散、匹配效率低、缺乏技能筛选机制等问题。为解决上述痛点，本项目拟设计并实现一个竞赛自由组队系统，为学生提供一个集中发布招募、精准匹配队友、在线沟通协作的组队平台，降低组队门槛，提高组队成功率。

#### （2）功能分析

1. **注册登录**：用户可进行注册与登录，系统采用加密存储，并实现多端互斥登录，自动将同一账号在其他设备挤下线以保障账号安全。

2. **个人名片**：新用户首次登录时系统将强制引导完善基础资料，用户可管理自己的专业、联系方式及自定义头像，其他用户可随时点击查看。

3. **技能标签**：用户可在个人资料中灵活选择自己的特长与兴趣标签，队长发布招募时也可预设所需技能要求，以实现精准定位。

4. **招募大厅**：队长可发布、隐藏、删除或停止项目招募，大厅卡片自动展示“招募中”或“已满员”状态，并实现页面无感局部自动刷新以展示最新动态。

5. **搜索筛选**：用户可通过预设的项目分类导航、输入关键词或组合筛选条件，在海量招募池中快速精准地定位心仪的项目或组队搭子。

6. **图文排版**：队长在编辑招募详情时支持使用Markdown语法进行排版，系统在前端自动渲染生成富文本展示页面，提升招募需求的专业度。

7. **智能推荐**：系统基于用户填写的专业背景及技能兴趣标签，通过匹配逻辑自动向用户个性化推送高度契合的潜在项目或互补队友。

8. **审批管理**：路人可一键提交入队申请，队长在后台进行双向审批，成团后队长亦可随时将违规成员移出队伍。

9. **消息红点**：系统全局监听用户的最新互动动态，通过网页右下角的弹窗提示及导航栏闪烁的红点和未读数量，实时提醒用户查看新消息。

10. **在线聊天**：系统提供私聊与项目群聊双向引擎，队长同意申请后自动拉群，用户可在聊天面板中无缝沟通并交换微信/QQ等外部联络信息，支持系统状态播报。

## 三、拟采用的实现技术

1. **后端技术**：采用 Python + FastAPI 框架，利用其异步特性和自动 API 文档生成能力，实现后端业务逻辑与接口开发。使用 Uvicorn 作为 Web 服务器。

2. **前端技术**：采用 Jinja2 模板引擎渲染页面，结合 HTML5 + CSS3 + 原生 JavaScript 实现交互功能，通过 Fetch API 完成异步数据请求。

3. **数据存储方案**：使用 SQLite 轻量级数据库进行数据持久化存储，无需额外安装配置，简化部署流程。

4. **开发运行环境**：Python 3.13 + VS Code / PyCharm，依赖通过 pip 管理。
